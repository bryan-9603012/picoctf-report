- **目標：** 根據提供的密文、金鑰提示與 Vigenère table，解出正確金鑰與 flag

## 二、題目概述

本題描述指出，一次性密碼本理論上是安全的，但若攻擊者掌握了金鑰，則加密內容就不再安全。題目已提供：

1. 加密後的字串：`UFJKXQZQUNB`
2. 已知對應明文：`SOLVECRYPTO`
3. 一份 Vigenère table

題目要求玩家利用這些資訊，推回加密所使用的金鑰，進而取得最終 flag。

## 三、分析目標

本次分析的主要目標如下：

1. 確認題目使用的加密方式
2. 根據已知明文與密文推回金鑰
3. 驗證每一個字元的對應關係
4. 說明常見誤判原因
5. 得出最終 flag

## 四、加密方式判定

題目提供的是一張標準的字母移位表，內容為每一列依序循環右移的 26x26 字母表。這是典型的 **Vigenère cipher table**。

Vigenère cipher 的基本規則如下：

- **加密：** 明文 + 金鑰 = 密文
- **解金鑰：** 密文 - 明文 = 金鑰

在表格使用上，一般採用：

- **列（row）= 金鑰字母**
- **欄（column）= 明文字母**
- **交會結果 = 密文字母**

因此，本題不是直接暴力破解，而是利用「已知明文攻擊」方式，把每一個位置的金鑰字母反推出來。

## 五、弱點判定

### 5.1 弱點描述

本題雖然不是在考傳統系統漏洞，但其核心弱點在於：

- 已知部分明文 `SOLVECRYPTO`
- 已知對應密文 `UFJKXQZQUNB`
- 題目還提供了完整的 Vigenère table

這使得攻擊者可以直接做逐字比對，反推出整段金鑰。

### 5.2 弱點類型

- Known-plaintext attack
- Vigenère key recovery

### 5.3 風險說明

若加密系統同時暴露：

1. 明文片段
2. 對應密文
3. 加密規則

則攻擊者就能直接回推出金鑰。  
本題正是透過這種方式，從 `SOLVECRYPTO` 與 `UFJKXQZQUNB` 還原出完整 key。

## 六、攻擊思路

已知：

- 明文：`SOLVECRYPTO`
- 密文：`UFJKXQZQUNB`

逐字計算：

`key = ciphertext - plaintext`

以第一個字元為例：

- 明文：`S`
- 密文：`U`

將字母轉為位移值：

- `S = 18`
- `U = 20`

所以：

- `20 - 18 = 2`
- 位移 2 對應字母 **C**

因此第一個金鑰字母是 **C**，不是 **M**。

若有人對出 **M**，通常代表把表格方向用反了，也就是把「列」和「欄」的角色顛倒。

將所有字母串接後得到：

成功還原出金鑰：

`CRYPTOISFUN`

## 十、成因分析

本題可以被解出的根本原因，在於 Vigenère cipher 並非真正的一次性密碼本。  
它的安全性依賴於：

1. 金鑰不可重建
2. 明文不可被攻擊者已知
3. 攻擊者無法輕易取得加密規則或對照表

但本題直接提供了：

- 密文
- 已知明文
- 對照表

因此實際上已足夠讓攻擊者逐字還原金鑰，完全不需要暴力破解。

## 十一、防禦建議

為避免此類問題，可採取以下措施：

### 11.1 避免重複或可推測金鑰

若金鑰可被明文與密文對照推出，加密就失去意義。

### 11.2 不要暴露已知明文

一旦攻擊者掌握某段明文與其密文配對，就可能回推出金鑰。

### 11.3 不應將古典密碼當作實務安全機制

Vigenère cipher 適合教學與競賽基礎訓練，但不適合現代安全用途。

## 十二、結論

本題核心在於辨識 Vigenère cipher，並利用已知明文 `SOLVECRYPTO` 和密文 `UFJKXQZQUNB` 逐字反推出金鑰。經過對照後可得：

`CRYPTOISFUN`

因此成功取得 flag：

`picoCTF{CRYPTOISFUN}`

另外，若在第一組 `S` 與 `U` 的對照中得到 `M`，通常代表表格方向使用錯誤；正確方式應為「密文減明文」，因此結果應為 **C**。
