# picoCTF 2022 - basic-mod2

## 基本資訊
- **題目名稱**: basic-mod2
- **平台 / 年份**: picoCTF 2022
- **分類**: Cryptography
- **難度**: Medium

## 題目描述
題目提供一份數字訊息檔案，並給出解密規則：

- 將每個數字對 41 取模
- 對結果求在 mod 41 下的 modular inverse（模反元素）
- 1–26 對應大寫英文字母 `A–Z`
- 27–36 對應十進位數字 `0–9`
- 37 對應底線 `_`

最後再將解出的內容包成 picoCTF flag 格式：

`picoCTF{decrypted_message}`

## 解題觀察
這題和 `basic-mod1` 的差別在於，不能把 `n % 41` 的結果直接拿去查表，而是還要多做一步 **modular inverse**。

也就是說，正確流程是：

1. 先算 `n % 41`
2. 再找該值在 mod 41 下的反元素
3. 最後再把反元素對照成字元

其中 modular inverse 的意思是：對某個數 `r`，找一個 `x` 使得：

`r * x ≡ 1 (mod 41)`

這個 `x` 才是最後要拿去查表的數值。

## 核心觀念
本題主要考查兩個觀念：

1. **模運算**
   - `n % 41` 表示除以 41 後的餘數
   - 這一步只是中間結果，不是最終對照值

2. **Modular Inverse**
   - 要找某個 `x`，使得 `r * x ≡ 1 (mod 41)`
   - 這個 `x` 才是最後拿去映射字元的值

## Python 範例
以下是可重現此題解法的簡單 Python 程式：

```python
nums = [104, 372, 110, 436, 262, 173, 354, 393, 351, 297, 241, 86, 262, 359, 256, 441, 124, 154, 165, 165, 219, 288, 42]

result = []

for n in nums:
    r = n % 41
    inv = pow(r, -1, 41)

    if 1 <= inv <= 26:
        result.append(chr(ord('A') + inv - 1))
    elif 27 <= inv <= 36:
        result.append(str(inv - 27))
    elif inv == 37:
        result.append('_')

decoded = ''.join(result)
print(decoded)
print(f"picoCTF{{{decoded}}}")
```

## 心得
這題比 `basic-mod1` 多了一層 modular inverse，因此如果只做到 `% 41` 就直接查表，會得到錯的答案。重點是理解模反元素的定義，並把它視為真正的映射輸入值。整體來說，這題是很典型的初階模運算與反元素練習。

## 重點整理
1. 先計算 `n % 41`
2. 再求該值在 mod 41 下的 modular inverse
3. 反元素才是最後查表用的數值
4. 1–26 對應 `A-Z`
5. 27–36 對應 `0-9`
6. 37 對應 `_`
