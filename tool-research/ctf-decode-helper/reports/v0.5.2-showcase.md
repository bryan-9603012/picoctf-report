# ctf-decode-helper v0.5.2 Showcase

## Overview

v0.5.2 focuses on noise cleanup for Top Results in heuristic-guided search. Building on v0.5.0/v0.5.1, this release further suppresses low-value noise while keeping the correct decode chain ranked first.

**Key improvements:**
- Stronger base64-like suppression (new `_is_strong_base64_like()`)
- Reverse penalty for noise chains without flags
- Noise family suppression for BASE64→CAESAR_SHIFT_n duplicates
- 9 new tests (total 74 tests, all passing)

## Heuristic-Guided Search Explanation

### Applicability Scoring (v0.5.0)
Each decoder computes an applicability score (0-100) before running:
- **0-9:** Not suitable or rarely useful (e.g., REVERSE on any text)
- **10-49:** Somewhat suitable (e.g., ROT13 on mixed text)
- **50-89:** Good candidate (e.g., Base64 on valid base64-like string)
- **90-100:** Strong match (e.g., BYTES_LITERAL_EXTRACT on `b'...'`)

Only decoders with applicability ≥ 10 are expanded in recursive mode.

### Transition Policy (v0.5.0+)
Adjusts applicability scores based on output characteristics:

| Output Type | Boosted Decoders | Suppressed Decoders |
|------------|------------------|---------------------|
| Bytes literal (`b'...'`) | BYTES_LITERAL_EXTRACT (90+) | ROT13/CAESAR (-30) |
| Base64-like | BASE64 (90+), BYTES_LITERAL_EXTRACT (85+) | ROT13/CAESAR (2 for strong) |
| Strong base64-like (v0.5.2) | BASE64 (95+) | ROT13/CAESAR (2), REVERSE (3) |
| A1Z26-like numbers | A1Z26 (85+) | Base64 (-10) |
| Substitution cipher text | ROT13/CAESAR (60+) | - |
| Reversible pattern (`}` prefix) | REVERSE (55+) | - |

### Noise Suppression (v0.5.2)

**1. Strong Base64-like Detection (`_is_strong_base64_like()`)**
- Alpha ratio ≥ 0.5 (vs 0.3 for regular)
- Digit ratio ≤ 0.3 (vs 0.5 for regular)
- Length ≥ 8 characters
- Proper padding (ends with `=` or length % 4 == 0)

**2. Reverse Penalty (`_apply_reverse_penalty()`)**
Chains containing REVERSE are penalized when:
- No flag found
- Output is still base64-like or random-like
- No keywords detected
- No structural improvement

Penalty: -100 score reduction

**3. Noise Family Suppression (`_suppress_noise_family()`)**
Same-family noise results (e.g., BASE64 → CAESAR_SHIFT_1, 2, 3...) are suppressed:
- Only top 2 results from each noise family kept at original score
- Rest penalized by -80 points

## Current Limitations

1. **No key-based crypto:** No AES, RSA, XOR (by design; postponed to future Crypto Mode)
2. **No image/packet/web analysis:** Text input only
3. **Reversed flags:** Not auto-detected (REVERSE decoder shows output, but flag detector needs human recognition)
4. **Applicability scoring:** Uses simple pattern matching; may miss edge cases
5. **Result dedup:** No semantic deduplication of identical decoded outputs across different chains
6. **Single-byte XOR:** Not implemented; only consider if a real challenge requires it
